Fundació Ave Maria · Design System · Iconografia

30 icones fets a mida per a la marca.

Sistema d'icones SVG dissenyat des del llenguatge geomètric d'Ave Maria. Traçat fi, cantonades arrodonides, geometria pura — coherent amb les 21 formes del sistema generatiu del isotip. Inclou guia d'ús accessible (WCAG 2.2 AA). Fes clic en qualsevol icona per copiar-ne el SVG.

05 · Aplicació · A11y

Ús accessible · de les icones 3 escenaris, 3 patrons

Una icona pot ser decorativa (acompanya text que ja explica l'acció), significativa (és l'únic indicador del que fa l'element) o amb funció (és un botó o enllaç sense text visible). Cada cas requereix un tractament diferent. Aquesta és la regla d'or per a no trencar mai l'accessibilitat amb iconografia.

Escenari 1
Decorativa
L'icona acompanya text que ja explica l'acció. L'icona repeteix el missatge visualment, però per al lector de pantalla és redundant. Cal amagar-la.
<button>
  <svg aria-hidden="true" focusable="false">…</svg>
  Fer un donatiu
</button>
Escenari 2
Significativa
L'icona és l'únic indicador d'un significat (un check verd que indica "actiu", una creu que indica "tancat"). Cal donar-li nom amb role="img" + aria-label.
<svg role="img"
     aria-label="Sol·licitud aprovada">…</svg>
Escenari 3
Botó/enllaç només icona
El botó NO té text visible (icona social, "tancar", lupa de cerca). El nom accessible ha d'estar al aria-label de l'element interactiu, i el SVG es marca com a decoratiu.
<a href="…"
   aria-label="Instagram">
  <svg aria-hidden="true">…</svg>
</a>

Contrast d'icones (1.4.11 AA)

Les icones són elements visuals no textuals: el seu contrast amb el fons ha de ser ≥3:1. Quan tenyim una icona amb un color de marca (taronja, coral…) sobre blanc, comprovem-ho.

Color de l'iconaSobre fons blancResultat
--am-tinta17.4:1PASS sempre
--am-color-success4.0:1PASS UI (≥3:1)
--am-color-danger5.1:1PASS UI
--am-taronja2.83:1FAIL · usar --am-taronja-ink per icones d'estat
--am-groc1.86:1FAIL · només decoratiu, mai com a icona d'estat
05 · Aplicació

Sprite SVG · format optimitzat

Un únic fitxer icons.svg amb totes les icones com a <symbol>. Al projecte web, es referencien per ID amb <use>. Resultat: descàrrega única, cau optimitzat, fàcil de mantenir.

Com utilitzar el sprite 
<!-- 1. Inclou el sprite a l'inici del <body> (visualment ocult) -->
<svg style="display:none">
  <symbol id="i-heart" viewBox="0 0 24 24">
    <path d="M12 20s…"/>
  </symbol>
  <!-- la resta de símbols -->
</svg>

<!-- 2. Per a cada icona, fes servir <use> -->
<svg class="am-icon am-icon--md">
  <use href="#i-heart"/>
</svg>

Tres formats per integrar el set al teu flux de treball: sprite SVG (web), JSON (Figma o build tools), CSS amb classes (web ràpida sense build).

Un únic fitxer SVG amb 30 <symbol>. Pesa menys de 8 KB. És el mètode recomanat per a la web — només es descarrega una vegada i les icones s'usen amb <use>.
icons-ave-maria.json — per a eines
Un JSON amb el nom, categoria, mida i path de cada icona. Útil per generar components React/Vue automàticament o per importar a Tokens Studio (Figma).
icons-ave-maria.html — pàgina de referència
Aquesta mateixa pàgina, lleugera, amb totes les icones i els seus codis SVG individuals. Comparteix-la amb el teu equip de desenvolupament.
08 · Anex

Accessibilitat · WCAG 2.2 AA aplicat al sistema d'icones

Tota la iconografia compleix els criteris d'accessibilitat WCAG 2.2 nivell AA. Aquest anex documenta com aquest set d'icones s'ha dissenyat i com s'ha de fer servir per garantir la conformitat. Per a una fundació social com Ave Maria, l'accessibilitat no és opcional: és la pròpia missió.

Compromís
WCAG 2.2 AA
Estàndard W3C, exigible legalment a la UE per entitats públiques (Directiva 2016/2102, RD 1112/2018) i recomanat per a qualsevol fundació o ONG. La iconografia és un element no-textual: la seva accessibilitat depèn molt del marcat semàntic.
Aplicat al disseny
Forma + color
Cada icona transmet el seu significat principalment per la forma, no pel color. Així, una persona daltònica reconeix igual un check-circle que un x-circle (1.4.1 AA, ús del color).
Aplicat al codi
Marcat semàntic
aria-hidden, role="img", aria-label, focusable="false": cada un té el seu lloc. La secció 05 d'aquesta pàgina explica quan usar cadascun.

Checklist · criteris coberts pel sistema

  • ✓1.1.1 (A)Text alternatiuCada SVG dins d'un context interactiu té el seu nom accessible. Els SVG decoratius es marquen amb aria-hidden="true". La galeria d'aquesta pàgina té cada tile com a <button> amb aria-label generat dinàmicament.
  • ✓1.3.3 (A)Sense dependència de la forma solaLes icones d'estat (success, danger…) sempre s'utilitzen acompanyades de text. Mai un check verd sol significa "èxit" — sempre amb text "Èxit: …".
  • ✓1.4.1 (A)Ús del colorEl color no és l'únic mitjà per transmetre informació. La forma de cada icona és distintiva: check-circle té un check, x-circle té una creu — un usuari daltònic els distingeix sense color.
  • ✓1.4.3 (AA)Contrast ≥4.5:1 per a text properQuan una icona acompanya text, el text manté contrast 4.5:1. Les icones d'estat utilitzen els tokens "ink" (versió enfosquida) garantint contrast accessible.
  • ✓1.4.11 (AA)Contrast d'elements UI ≥3:1Les icones són elements UI no-textuals i compleixen 3:1 mínim sobre el seu fons. Veure taula a la secció 05.
  • ✓1.4.5 (AA)SVG escalable, no imatge rasterEl format SVG és vectorial: l'usuari pot fer zoom fins al 400% sense pèrdua de qualitat. Mai usar PNG d'icones — sempre SVG.
  • ✓2.1.1 (A)Operable amb teclatCada tile de la galeria és un <button> que respon a Enter i Espai. Tab hi navega. El sprite SVG es pot copiar amb teclat.
  • ✓2.4.4 (A)Propòsit dels enllaços/botons clarCada tile té com a aria-label "Copiar el SVG de la icona X". Cap acció amb destí ambigu.
  • ✓2.4.7 (AA)Focus visibleOutline sòlid de 3 px amb offset al rebre el focus en qualsevol tile o botó. Sobre fons fosc, blanc.
  • ✓2.5.5 (AAA) / 2.5.8 (AA)Mida tàctilCada tile fa ≥110 px d'alt i ≥140 px d'ample. Molt per sobre dels 24×24 mínims de WCAG 2.5.8.
  • ✓2.3.3 (AAA)Reducció de movimentSi l'usuari té configurat prefers-reduced-motion: reduce al seu sistema, totes les transicions hover de les tiles es desactiven.
  • ✓4.1.2 (A)Nom · Rol · ValorCada tile interactiva exposa el seu nom (aria-label), el seu rol (button) i el seu estat (is-copied després de prémer).
  • ✓4.1.3 (AA)Missatges d'estatEl toast "✓ SVG copiat" usa una live region implícita perquè el lector de pantalla anuncïi que l'acció s'ha completat sense necessitat que l'usuari ho descobreixi.

Tres regles inviolables

  1. Mai una icona sense decisió conscient sobre el seu rol accessible. Cada vegada que afegeixes una icona, has de decidir: és decorativa? significativa? funcional? La resposta determina el marcat.
  2. Mai només color per comunicar un estat. Un check verd no és "OK" sense la paraula "OK" o un context evident. Una creu vermella no és "error" sense text d'error.
  3. Mai una icona-link per sota dels 24×24 px tocables. Si la icona visualment és petita (16-20 px), envolta-la amb padding fins que la zona clicable arribi a 24×24 mínim. Recomanat 44×44.
